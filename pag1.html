MAS INFORMACION

1.1. Definición e importancia de la simulación en la Ingeniería

Definición: La simulación es una técnica que permite representar el comportamiento de un sistema real a través de un modelo computacional. Es especialmente útil cuando el sistema es complejo, costoso o peligroso de experimentar en la vida real. Importancia en Ingeniería: Permite analizar y optimizar sistemas sin necesidad de implementarlos físicamente. Reduce riesgos, costos y tiempo de experimentación. Facilita la toma de decisiones basada en modelos virtuales. Es crucial en áreas como manufactura, transporte, telecomunicaciones, logística, entre otras.

1.2. Conceptos básicos de la simulación

Sistema: Conjunto de elementos interrelacionados que cumplen una función. Modelo: Representación simplificada del sistema (puede ser físico, matemático o computacional). Evento: Suceso que cambia el estado del sistema. Estado: Conjunto de variables que describen un sistema en un momento dado. Variables:De estado: describen la situación actual del sistema. De decisión: pueden controlarse para optimizar resultados. Tiempo de simulación: Representación del avance del tiempo en el modelo. Reloj de simulación: Controla la progresión del tiempo en el modelo.

1.3. Metodología de la simulación

La metodología de simulación implica los siguientes pasos: Definición del problema. Formulación del modelo. Recolección de datos del sistema real. Construcción del modelo en software. Verificación del modelo (¿Funciona correctamente?). Validación del modelo (¿Representa la realidad?). Ejecutar simulaciones. Analizar los resultados. Documentar y tomar decisiones.

1.4 Estructura y etapas de un estudio de simulación

Un estudio de simulación está compuesto por varias etapas que se desarrollan de forma estructurada. Primero, se formula el problema a resolver, definiendo con claridad cuáles son las preguntas que se quieren contestar mediante la simulación. Luego, se establece el sistema a estudiar y se determinan sus límites, es decir, qué aspectos se incluirán en el modelo y cuáles quedarán fuera. Después se recolectan los datos necesarios del sistema real, lo que implica mediciones, entrevistas o análisis de documentación. Con esos datos, se elabora un modelo conceptual que define las relaciones entre las partes del sistema. El siguiente paso es convertir ese modelo en un modelo computacional, utilizando software de simulación. Una vez implementado, se procede a verificar que el modelo funcione correctamente (verificación) y a validar que represente fielmente al sistema real (validación).

1.5 Etapas de un proyecto de simulación

Un proyecto de simulación avanza por una serie de etapas que permiten estructurar el trabajo de forma eficiente. La primera etapa es el análisis del sistema, donde se estudia el contexto y se identifica el problema a resolver, definiendo también los objetivos del proyecto. Luego, en la etapa de diseño del modelo, se seleccionan las variables que se van a considerar, se identifican los eventos relevantes y se establecen las relaciones entre ellos. A continuación, se implementa el modelo en un entorno computacional, utilizando herramientas como Arena, Simul8, FlexSim o lenguajes de programación como Python o Java. Después se procede a la verificación y validación del modelo, para asegurar que funcione correctamente y represente bien al sistema real. Una vez validado, se realizan simulaciones en diferentes condiciones, lo que permite observar el comportamiento del sistema bajo escenarios alternativos. Con los resultados de la simulación, se evalúan posibles soluciones o mejoras y se elaboran recomendaciones. Finalmente, se elabora un informe que presenta todo el trabajo realizado y los resultados obtenidos.

1.6 Elementos básicos de un simulador de eventos discretos

Un simulador de eventos discretos se basa en una estructura interna que incluye varios elementos fundamentales. El primero es el reloj de simulación, que controla la progresión del tiempo dentro del modelo y avanza de un evento al siguiente. Otro elemento importante es la agenda de eventos futuros, que es una lista ordenada cronológicamente con los próximos eventos que deben ejecutarse. Cada evento tiene una rutina asociada que especifica las acciones que deben realizarse cuando ese evento ocurre, como actualizar variables o generar nuevos eventos.

1.7 Ventajas y desventajas de la simulación

La simulación ofrece varias ventajas importantes. En primer lugar, permite analizar el comportamiento de un sistema sin necesidad de construirlo físicamente, lo cual ahorra tiempo y dinero. También permite estudiar diferentes escenarios y condiciones sin afectar el funcionamiento real del sistema, lo cual es especialmente útil cuando se trata de procesos críticos o peligrosos. La simulación ayuda a identificar cuellos de botella, evaluar alternativas y optimizar procesos antes de tomar decisiones en la vida real. Además, es una herramienta flexible, que puede adaptarse a diferentes tipos de sistemas y situaciones. Sin embargo, también tiene desventajas.

Número pseudoaleatorio

Un número pseudoaleatorio es un número que parece aleatorio, pero en realidad ha sido generado por un algoritmo determinista. Esto significa que, si se conocen las condiciones iniciales (como la semilla), se puede predecir la secuencia completa. Estos números son ampliamente utilizados en simulación, criptografía, videojuegos, estadísticas y pruebas de sistemas, ya que permiten generar datos que simulan el comportamiento aleatorio. Se les llama "pseudo" porque no son verdaderamente aleatorios, sino que imitan la aleatoriedad mediante fórmulas matemáticas.

Generación de variables aleatorias

La generación de variables aleatorias consiste en producir valores numéricos que siguen una distribución de probabilidad específica (como uniforme, normal, exponencial, Poisson, etc.), a partir de números pseudoaleatorios uniformes. Dado que las computadoras no pueden generar verdaderos números aleatorios, se usan algoritmos que generan números pseudoaleatorios uniformes entre 0 y 1. A partir de ellos, se generan variables aleatorias con otras distribuciones mediante transformaciones matemáticas.

3.1 Conceptos básicos

Las variables aleatorias representan resultados de fenómenos inciertos. Pueden ser discretas o continuas y se usan para modelar eventos aleatorios.

3.2 Variables aleatorias discretas

Toman valores específicos y contables, como el número de hijos en una familia. Se describen con funciones de probabilidad.

3.3 Variables aleatorias continuas

Pueden asumir infinitos valores dentro de un intervalo, como la estatura. Se representan con funciones de densidad.

3.4 Métodos para generar variables aleatorias

Técnicas usadas en simulación para obtener valores aleatorios con una distribución específica.

3.4.1 Método de la transformada inversa

Convierte un número aleatorio uniforme en otro con la distribución deseada usando la función inversa de la distribución acumulada.

3.4.2 Método de convolución

Suma variables aleatorias independientes para obtener una nueva variable con una distribución específica, como la normal o gamma.

3.4.3 Método de composición

Combina varias distribuciones conocidas en proporciones definidas para generar una nueva variable aleatoria.

3.5 Procedimientos especiales

Métodos diseñados para distribuciones complejas, como la normal, exponencial o Poisson, que requieren técnicas específicas para su generación.

3.6 Pruebas estadísticas

Evalúan si los datos generados siguen la distribución deseada. Incluyen pruebas como Chi-cuadrado, Kolmogorov-Smirnov y rachas.

Lenguaje de simulacion

Podras ver los digerentes lenguajes de simulacion y mas hacerca de ellos

1. SIMULINK (de MATLAB)

Descripción: Es un entorno de simulación basado en diagramas de bloques que permite modelar sistemas dinámicos y de control.
Cómo funciona: El usuario arrastra y conecta bloques que representan funciones matemáticas, entradas, salidas, sensores, actuadores, etc.
Aplicaciones: Control automático, electrónica, robótica, procesamiento de señales.

2. ARENA

Descripción: Es un software de simulación de eventos discretos utilizado para analizar procesos industriales y de servicios.
Cómo funciona: El usuario crea un flujo de entidades (como personas, productos o documentos) a través de procesos y recursos.
Aplicaciones: Producción, logística, centros de atención al cliente, hospitales.

Mas Informacion

3. VHDL (VHSIC Hardware Description Language)

Descripción: Es un lenguaje de programación especializado para describir y simular el comportamiento de circuitos digitales.
Cómo funciona: Se escribe código que representa cómo se comportan compuertas, registros y circuitos lógicos. Se puede simular en software o sintetizar para hardware real (como FPGA).
Aplicaciones: Diseño de microprocesadores, sistemas embebidos, electrónica digital.
